Giriş: JavaScript Nedir ve Nasıl Çalışır?

JavaScript, web sayfalarını interaktif ve dinamik hale getiren bir programlama dilidir. HTML ve CSS ile birlikte web geliştirmenin üç temel taşından biridir.

Ev analojimizde, HTML iskelet, CSS dekorasyon ise JavaScript o evin beyni ve elektrik tesisatıdır. Butona basınca ışığın yanması, form gönderme, animasyonlar ve sunucudan veri çekme gibi tüm "eylemler" JavaScript ile yapılır.

JS Kodu Sayfaya Nasıl Eklenir?

Tıpkı CSS gibi, 3 yolu vardır. Ve yine tıpkı CSS gibi, en iyi yöntem "Harici" olanıdır.

  1. Harici (External) - En İyi Yöntem: Kodlar script.js gibi ayrı bir dosyaya yazılır.
  2. Dahili (Internal): Kodlar, <script> ... </script> etiketleri arasına yazılır.
  3. Satır İçi (Inline): Kodlar, onclick="..." gibi HTML niteliklerine yazılır. (Kesinlikle tavsiye edilmez).

ÖNEMLİ KURAL: <script> Nereye Konur?

CSS <link> etiketinin aksine, <script src="script.js"></script> etiketi genellikle <head> içine **konulmaz**.

Bunun yerine, <body> etiketinin **kapanışından hemen önceye**, yani sayfanın en altına konur. 

  ... (tüm HTML içeriği) ...
    <script src="script.js"></script>
</body>
</html>

Neden? Çünkü JS, HTML elementlerini (buton, başlık vb.) manipüle edecektir. Tarayıcının, JS kodunu çalıştırmadan önce o HTML elementlerini ekrana çizmiş (parse etmiş) olduğundan emin olmak isteriz. Aksi halde JS, var olmayan bir butonu seçmeye çalışır ve hata verir.

İlk Aracımız: `console.log()`

console.log(), bir geliştiricinin en iyi dostudur. Değişkenlerin değerini veya kodun çalışıp çalışmadığını görmek için tarayıcının "Geliştirici Konsolu"na (F12 > Console) mesaj yazdırmanızı sağlar.

console.log("Merhaba Dünya!"); // F12 > Console sekmesinde bunu göreceksin.
console.log(4 + 4); // Konsola "8" yazar.

Temeller: Değişkenler ve Veri Türleri

Değişkenler: `let`, `const` (ve `var`)

Değişkenler, verileri depolamak için kullanılan isimlendirilmiş kaplardır.

  • let: Değeri daha sonra **değiştirilebilen** değişkenler için kullanılır. (Modern var'dır).
  • const: Değeri **sabit** kalacak, hiç değişmeyecek değişkenler (constants) için kullanılır.
  • var: Eski JavaScript değişken tanımlama yöntemidir. Kapsam (scope) sorunları nedeniyle modern JS'te artık kullanılması **tavsiye edilmez**. Her zaman let veya const kullanın.
// Değeri değişebilen bir değişken
let kullaniciYasi = 25;
kullaniciYasi = 26; // Geçerli.

// Sabit bir değişken
const dogumYili = 1999;
// dogumYili = 2000; // Hata! const ile tanımlanan değişken değiştirilemez.

const ad = "Ahmet";

Temel Veri Türleri (Primitive Types)

  • String (Metin): Tırnak işaretleri içindeki metinlerdir.
    const isim = "Ayşe"; (Çift veya 'tek' tırnak kullanılabilir).
  • Number (Sayı): Hem tam sayılar (10) hem de ondalıklı sayılar (3.14) için kullanılır.
    let puan = 100;
  • Boolean (Mantıksal): Sadece iki değeri olabilir: true (doğru) veya false (yanlış).
    const girisYaptiMi = true;
  • undefined: Bir değişkene değer atanmadığında varsayılan değeridir.
    let sepet; // Değeri undefined
  • null: "Boş" veya "hiçbir değer yok" anlamına gelir. undefined'dan farklı olarak, bu "bilinçli" bir boşluktur.
    let seciliKullanici = null;

Operatörler

Değişkenler ve değerler üzerinde işlem yapmak için kullanılırlar.

1. Aritmetik Operatörler

+ (Toplama), - (Çıkarma), * (Çarpma), / (Bölme), % (Mod - Bölümden kalan).

let sonuc = 10 + (5 * 2); // sonuc = 20
let kalan = 10 % 3; // kalan = 1 (10/3'ten kalan 1'dir)

// String birleştirme
let tamAd = "Ahmet" + " " + "Yılmaz"; // "Ahmet Yılmaz"

2. Karşılaştırma Operatörleri (ÇOK ÖNEMLİ)

İki değeri karşılaştırır ve true veya false (Boolean) bir sonuç döndürür.

  • > (Büyük), < (Küçük), >= (Büyük veya eşit), <= (Küçük veya eşit)
  • == (Gevşek Eşitlik): **Kullanmamaya çalışın!** Değerleri karşılaştırırken tiplerini zorla dönüştürür.
    10 == "10" // true (Bu genellikle istenmez)
  • === (Katı Eşitlik): **Her zaman bunu kullanın!** Hem değeri hem de veri tipini karşılaştırır.
    10 === "10" // false (Çünkü biri number, biri string)
    10 === 10 // true
  • != (Gevşek Eşit Değil), !== (Katı Eşit Değil - **Bunu kullanın!**)

3. Mantıksal Operatörler

Genellikle if koşullarında birden fazla durumu birleştirmek için kullanılır.

  • && (AND - Ve): Her iki koşul da true ise true döner.
    (yas > 18 && ehliyetVarMi)
  • || (OR - Veya): Koşullardan en az biri true ise true döner.
    (kullaniciAdi === "admin" || kullaniciAdi === "moderator")
  • ! (NOT - Değil): Bir boolean değerin tersini alır.
    !girisYaptiMi // Eğer true ise false, false ise true yapar

Koşullar (If/Else, Switch)

Kodun, belirli bir koşulun true veya false olmasına bağlı olarak farklı yollar izlemesini sağlar.

`if`, `else if`, `else`

En yaygın kullanılan koşul yapısıdır.

const saat = 14;

if (saat < 12) {
  console.log("Günaydın!");
} else if (saat < 18) {
  console.log("Tünaydın!");
} else {
  console.log("İyi akşamlar!");
}
// Konsola "Tünaydın!" yazar.

`switch`

Bir değişkenin alabileceği çok sayıda spesifik değere karşı kontrol yapmak için if/else if zincirine temiz bir alternatiftir.

const gun = "Salı";

switch (gun) {
  case "Pazartesi":
    console.log("Haftanın ilk günü.");
    break; // break: bloğu terk et
  case "Salı":
  case "Çarşamba":
    console.log("Hafta ortası.");
    break;
  case "Cuma":
    console.log("Hafta sonu yaklaşıyor!");
    break;
  default:
    console.log("Normal bir gün.");
}
// Konsola "Hafta ortası." yazar.

Döngüler (Loops)

Bir kod bloğunu belirli bir koşul sağlandığı sürece veya belirli bir sayıda tekrar tekrar çalıştırmanızı sağlar.

`for` Döngüsü

En yaygın döngü türüdür. Başlangıcı, bitiş koşulu ve artış miktarı bellidir.

// 0'dan 4'e kadar (5 dahil değil) sayıları yazdır:
for (let i = 0; i < 5; i++) {
  console.log(i);
}
// Konsola yazar:
// 0
// 1
// 2
// 3
// 4

`while` Döngüsü

Sadece bir koşul true olduğu sürece çalışmaya devam eder. Koşulun döngü içinde false'a dönmesini sağlamak sizin sorumluluğunuzdadır, yoksa "sonsuz döngü" oluşur.

let sayac = 0;
while (sayac < 3) {
  console.log("Sayac: " + sayac);
  sayac++; // Koşulu false yapmak için sayacı artır
}
// Konsola yazar:
// Sayac: 0
// Sayac: 1
// Sayac: 2

Fonksiyonlar (ve Arrow Functions)

Fonksiyonlar, belirli bir görevi yerine getiren, yeniden kullanılabilir kod bloklarıdır. Kod tekrarını önlerler.

1. Fonksiyon Bildirimi (Function Declaration)

Standart, isimlendirilmiş fonksiyon tanımı.

// Fonksiyonu TANIMLA
function selamVer(isim) {
  console.log("Merhaba, " + isim + "!");
}

// Fonksiyonu ÇAĞIR
selamVer("Ayşe"); // Konsola "Merhaba, Ayşe!" yazar.
selamVer("Fatma"); // Konsola "Merhaba, Fatma!" yazar.

`return` Anahtar Kelimesi

Bir fonksiyondan bir değer "döndürmek" için kullanılır. console.log sadece konsola yazar, return ise değeri hesaplamanın yapıldığı yere geri gönderir.

function topla(sayi1, sayi2) {
  return sayi1 + sayi2;
}

const toplam = topla(5, 3);
console.log(toplam); // Konsola 8 yazar.

2. Arrow Functions (ES6) (Çok Önemli)

Modern JavaScript'te fonksiyon yazmanın daha kısa ve temiz bir yoludur. Özellikle `forEach`, `map` gibi metotlar içinde çok kullanılırlar.

// Geleneksel Yöntem (Function Expression)
const karesiniAl = function(sayi) {
  return sayi * sayi;
}

// Arrow Function Yöntemi
const karesiniAlArrow = (sayi) => {
  return sayi * sayi;
}

// Arrow Function (Tek satırlık return için daha kısa yazım)
const karesiniAlKisa = (sayi) => sayi * sayi;

console.log(karesiniAlKisa(4)); // Konsola 16 yazar.

Objeler (Objects)

Objeler, birbiriyle ilişkili verileri ve fonksiyonları (metotları) bir arada tutan koleksiyonlardır. anahtar: değer (key: value) çiftlerinden oluşurlar.

Obje Literali

Obje oluşturmanın en yaygın yoludur.

const kullanici = {
  // Anahtarlar (Keys)   // Değerler (Values)
  ad: "Ali",
  soyad: "Veli",
  yas: 30,
  girisYaptiMi: true,
  favoriRenkler: ["mavi", "yeşil"], // Değer bir dizi olabilir

  // Metot (Obje içindeki fonksiyon)
  tamAdiniGetir: function() {
    // 'this' anahtar kelimesi, içinde bulunduğu objeyi (kullanici) temsil eder.
    return this.ad + " " + this.soyad;
  }
};

Özelliklere Erişme

İki yolu vardır: Nokta Gösterimi (Dot Notation) ve Köşeli Parantez Gösterimi (Bracket Notation).

// 1. Nokta Gösterimi (En yaygın)
console.log(kullanici.ad); // "Ali"
console.log(kullanici.favoriRenkler[0]); // "mavi"

// Metodu çağırma
console.log(kullanici.tamAdiniGetir()); // "Ali Veli"

// 2. Köşeli Parantez Gösterimi (Anahtar bir değişkende tutuluyorsa zorunludur)
console.log(kullanici["yas"]); // 30

const secim = "soyad";
console.log(kullanici[secim]); // "Veli" (kullanici.secim çalışmazdı)

Diziler (Arrays)

Diziler, birden fazla değeri tek bir değişken altında sıralı bir listede tutmamızı sağlar. Diziler teknik olarak özel bir obje türüdür.

// Diziler 0 (sıfır) indis numarası ile başlar
const meyveler = ["Elma", "Armut", "Muz"];

Dizi Özellikleri ve Temel Metotlar

  • meyveler[0]: Dizinin ilk elemanına erişir ("Elma").
  • meyveler[1] = "Çilek";: Dizinin ikinci elemanını günceller.
  • meyveler.length: Dizinin eleman sayısını verir (3).
  • meyveler.push("Kiraz"): Dizinin **sonuna** yeni bir eleman ekler.
  • meyveler.pop(): Dizinin **sonundaki** elemanı çıkarır.

Modern ve Güçlü Dizi Metotları (ES6+)

Bu metotlar, döngü yazmak yerine veriyi dönüştürmek ve filtrelemek için kullanılır. Orijinal diziyi **değiştirmezler**, yeni bir dizi döndürürler.

`forEach` (Döngü)

Dizideki her bir eleman için bir fonksiyon çalıştırır. (for döngüsünün modern halidir).

meyveler.forEach(function(meyve, indis) {
  console.log(indis + ": " + meyve);
});
// VEYA Arrow Function ile:
meyveler.forEach((meyve) => console.log(meyve));

`map` (Dönüştürme)

Dizideki her bir elemanı alıp, onu *dönüştürerek* **yeni bir dizi** oluşturur.

const sayilar = [1, 2, 3, 4];
const ikiKati = sayilar.map((sayi) => sayi * 2);
// ikiKati = [2, 4, 6, 8]
// (orijinal 'sayilar' dizisi değişmedi)

`filter` (Filtreleme)

Dizideki elemanlardan sadece *koşulu sağlayanları* alıp **yeni bir dizi** oluşturur.

const sayilar = [1, 2, 3, 4, 5, 6];
const ciftSayilar = sayilar.filter((sayi) => sayi % 2 === 0);
// ciftSayilar = [2, 4, 6]

DOM: Element Seçme

JavaScript'in tarayıcıdaki gücü, HTML elementlerini (DOM - Document Object Model) seçebilmesinden ve değiştirebilmesinden gelir.

Modern Yöntem: `querySelector`

CSS seçicileri kullanarak bir element seçmenin en modern ve esnek yoludur. Sayfada bulduğu **ilk** eşleşmeyi döndürür.

// ID'ye göre seç
const baslik = document.querySelector("#ana-baslik");

// Sınıfa göre seç
const ilkKart = document.querySelector(".kart");

// Etikete göre seç
const ilkParagraf = document.querySelector("p");

// Daha karmaşık bir seçici
const ilkKartinResmi = document.querySelector(".kart img");

Modern Yöntem: `querySelectorAll`

querySelector gibidir ancak eşleşen **tüm** elementleri bir "NodeList" (diziye benzer) olarak döndürür.

// Sayfadaki TÜM kartları seç
const tumKartlar = document.querySelectorAll(".kart");

// NodeList üzerinde forEach ile dönebiliriz:
tumKartlar.forEach((kart) => {
  kart.style.border = "1px solid red"; // Tüm kartlara kırmızı kenarlık ekle
});

Eski Yöntemler (Hala Yaygın)

  • document.getElementById("ana-baslik"): Sadece ID ile seçer (Hızlıdır).
  • document.getElementsByClassName("kart"): Sınıfa göre seçer (Canlı bir HTMLCollection döndürür, forEach doğrudan çalışmaz).
  • document.getElementsByTagName("p"): Etikete göre seçer.

Tavsiye: Genellikle querySelector ve querySelectorAll kullanmak en temizidir.

DOM: Element Manipülasyonu

Elementleri seçtikten sonra onları değiştirebiliriz.

İçerik Değiştirme: `textContent` ve `innerHTML`

  • .textContent: Sadece metin içeriğini alır veya değiştirir. Güvenlidir.
  • .innerHTML: Elementin içindeki HTML kodunu alır veya değiştirir. Kullanıcıdan alınan veriyi buraya basmak XSS açıkları oluşturabileceğinden **tehlikeli** olabilir.
const baslik = document.querySelector("#ana-baslik");
baslik.textContent = "Yeni Harika Başlık!"; // Sadece metni değiştirir

const icerik = document.querySelector(".icerik");
icerik.innerHTML = "<h2>HTML Eklendi</h2><p>Bu bir paragraf.</p>";

Stil Değiştirme: `.style`

Elementlerin CSS özelliklerini doğrudan değiştirir. (Genellikle stil için class ekleyip/çıkarmak daha iyidir).

baslik.style.color = "blue";
baslik.style.backgroundColor = "#f0f0f0"; // CSS'teki "background-color" burada "backgroundColor" olur

Sınıf (Class) Değiştirme: `.classList` (En İyi Yöntem)

Stil değiştirmenin en temiz yolu, stilleri CSS'te tanımlayıp (örn: .aktif, .dark-mode) JavaScript ile bu sınıfları ekleyip çıkarmaktır.

  • .classList.add("aktif"): Sınıf ekler.
  • .classList.remove("aktif"): Sınıfı kaldırır.
  • .classList.toggle("aktif"): Sınıf yoksa ekler, varsa kaldırır (Aç/Kapa için harikadır).
// 

const menu = document.querySelector(".menu");
menu.classList.add("acik"); // 

menu.classList.toggle("acik"); //

Element Oluşturma ve Ekleme

// 1. Yeni bir 
  •  elementi oluştur
const yeniOge = document.createElement("li");

// 2. İçeriğini ayarla
yeniOge.textContent = "Yeni Liste Elemanı";

// 3. Mevcut bir listeyi seç
const liste = document.querySelector("ul");

// 4. Yeni elementi listenin sonuna ekle
liste.append(yeniOge); // (append modern yöntemdir, appendChild da kullanılır)

    • Olaylar (Events)

    Olaylar, kullanıcının yaptığı eylemlerdir (butona tıklama, fareyi gezdirme, klavyeden bir tuşa basma vb.). JavaScript ile bu olayları "dinleyebilir" ve olay gerçekleştiğinde bir kod bloğunu çalıştırabiliriz.

    `addEventListener` (En İyi Yöntem)

    Bir elemente bir olay dinleyici eklemenin modern ve esnek yoludur.

    // 1. Butonu seç
const buton = document.querySelector("#uyari-butonu");

// 2. 'click' (tıklama) olayını dinle
buton.addEventListener("click", function() {
  // 3. Olay gerçekleştiğinde bu fonksiyon çalışır
  alert("Butona tıklandı!");
});

// Arrow function ile daha kısa:
buton.addEventListener("click", () => {
  console.log("Tıklandı!");
});

    Yaygın Olay Türleri

    • click: Tıklama.
    • mouseover / mouseout: Fare üzerine gelme / çekilme.
    • keydown / keyup: Klavyede tuşa basma / bırakma.
    • input: Bir giriş alanının (<input>, <textarea>) değeri değiştiğinde.
    • submit: Bir form (<form>) gönderildiğinde.

    `event` Objesi ve `preventDefault()`

    Olay gerçekleştiğinde, çalışan fonksiyona otomatik olarak bir event (olay) objesi parametre olarak gönderilir.

    event.preventDefault(), tarayıcının o olay için varsayılan davranışını (örn: forma tıklayınca sayfanın yenilenmesi, linke tıklayınca başka sayfaya gitmesi) engeller.

    const form = document.querySelector("form");

form.addEventListener("submit", (event) => {
  event.preventDefault(); // Tarayıcının formu gönderip sayfayı yenilemesini ENGELLE

  console.log("Form gönderildi ama sayfa yenilenmedi.");
  // Burada form verilerini alıp kendimiz işleyebiliriz.
});

    ES6+ Modern JS Özellikleri

    ES6 (ECMAScript 2015), JavaScript'e devrim niteliğinde yenilikler getirdi. let/const, Arrow Functions ve map/filter dışında en çok kullanacaklarınız:

    Template Literals (Şablon Dizgeleri)

    Ters tırnak (`` - backtick) kullanarak, metin (string) içine ${...} söz dizimiyle kolayca değişken eklemenizi sağlar. (+ ile birleştirmeye son!)

    const ad = "Ayşe";
const yas = 28;

// Eski Yöntem:
const mesajEski = "Merhaba, benim adım " + ad + " ve ben " + yas + " yaşındayım.";

// Modern Yöntem (Template Literal):
const mesajYeni = `Merhaba, benim adım ${ad} ve ben ${yas} yaşındayım.`;

    Destructuring (Yapı Bozma)

    Objelerden veya dizilerden verileri çekip doğrudan değişkenlere atamanın kısa yoludur.

    // Obje Destructuring
const kullanici = {
  id: 1,
  kullaniciAdi: "coder123",
  email: "test@test.com"
};
// 'id' ve 'email'i objeden çekip değişken yap:
const { id, email } = kullanici;
console.log(id); // 1
console.log(email); // "test@test.com"

// Dizi Destructuring
const renkler = ["kırmızı", "yeşil", "mavi"];
const [ilkRenk, ikinciRenk] = renkler;
console.log(ilkRenk); // "kırmızı"

    Spread Operatörü (...)

    Bir dizinin veya objenin tüm elemanlarını "yayarak" yeni bir diziye/objeye kopyalamayı sağlar.

    const temelRenkler = ["kırmızı", "yeşil", "mavi"];
const tumRenkler = [...temelRenkler, "sarı", "mor"];
// tumRenkler = ["kırmızı", "yeşil", "mavi", "sarı", "mor"]

const kullaniciTemel = { ad: "Zeynep" };
const kullaniciTam = { ...kullaniciTemel, yas: 22 };
// kullaniciTam = { ad: "Zeynep", yas: 22 }

    Asenkron JS (fetch, async/await)

    Bu ileri düzey bir konudur ancak modern web'in temelidir. JavaScript varsayılan olarak "senkron" (sıralı) çalışır. Ancak bir sunucudan veri çekmek (API isteği) gibi zaman alan işlemler, tüm kodun kilitlenmesine neden olur.

    Asenkron (eşzamansız) programlama, bu zaman alan işlemin arka planda tamamlanmasını beklerken, kodun geri kalanının çalışmaya devam etmesini sağlar.

    fetch ile Veri Çekme (Promise Tabanlı)

    fetch, tarayıcıda sunuculardan (API'lardan) veri çekmek için kullanılan modern fonksiyondur. "Promise" (söz) döndürür; yani işlemin gelecekte bir zamanda tamamlanacağına dair bir söz.

    En Temiz Yöntem: `async / await`

    async/await, Promise'leri (sözleri) senkron (sıralı) kod gibi temiz bir şekilde yönetmemizi sağlayan modern bir söz dizimidir.

    1. Fonksiyonun başına async yazılır.
    2. Zaman alacak işlemin (fetch gibi) başına await yazılır.
    3. Kod, o satırın bitmesini bekler ve sonra devam eder.
    4. Hata yakalamak için try...catch bloğu kullanılır.
    // Bir API'dan rastgele kullanıcı verisi çeken bir fonksiyon

async function kullaniciGetir() {
  try {
    // 1. Await: API isteğinin bitmesini bekle
    const response = await fetch('https://jsonplaceholder.typicode.com/users/1');

    // 2. Await: Gelen cevabı JSON'a çevirmeyi bekle
    const kullanici = await response.json();

    // 3. Veriyi kullan
    console.log(kullanici.name); // Konsola "Leanne Graham" yazar
    document.querySelector("#ana-baslik").textContent = kullanici.name;

  } catch (hata) {
    // 4. Eğer bir ağ hatası olursa burası çalışır
    console.error("Veri çekilemedi:", hata);
