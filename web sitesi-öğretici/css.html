Giriş: CSS Nedir?

CSS (Cascading Style Sheets - Basamaklı Stil Şablonları), HTML elementlerinin ekranda nasıl görüneceğini belirleyen bir stil dilidir. Web sayfalarının görsel tasarımını ve düzenini oluşturmak için kullanılır.

Ev inşa etme analojimizi hatırlarsak: HTML iskeletti, CSS ise o iskeletin duvar rengi, mobilyaları, parke türü, kısacası tüm estetik ve dekorasyonuydu.

"Cascading" (Basamaklı) Ne Anlama Geliyor?

CSS'in en temel ve en güçlü özelliği "basamaklı" olmasıdır. Bu, stillerin bir öncelik sırasına göre uygulandığı anlamına gelir. Bir element için birden fazla stil kuralı tanımlanmışsa, tarayıcı hangisinin daha "spesifik" (belirgin) veya "önemli" olduğuna karar verir ve onu uygular. Bu sayede genel bir kural (örn: "tüm paragraflar mavi olsun") yazıp, daha sonra özel bir kural (örn: "ama ana sayfadaki bu paragraf kırmızı olsun") ile onu ezebilirsiniz.

CSS olmadan web, sıralı metin yığınlarından ibaret olurdu. CSS, web'e sanatı ve tasarımı getiren teknolojidir.

CSS Sayfaya Nasıl Eklenir? (3 Yöntem)

HTML belgelerimize stil eklemenin üç temel yolu vardır. En iyiden en kötüye doğru sıralanmışlardır:

1. Harici (External) CSS (En İyi Yöntem)

Tüm CSS kodlarımızı style.css gibi ayrı bir .css dosyasına yazarız ve bu dosyayı <link> etiketi ile HTML sayfamızın <head> bölümüne bağlarız.

Neden En İyisi?

  • Merkezi Yönetim: Yüzlerce sayfalık bir sitenin stilini tek bir dosyadan değiştirebilirsiniz.
  • Temiz Kod: HTML (yapı) ve CSS (görünüm) birbirinden tamamen ayrılır.
  • Hız (Önbellekleme): Tarayıcı, style.css dosyasını bir kez indirir ve diğer sayfalara geçtiğinizde tekrar indirmek yerine önbellekten kullanır.

<!-- index.html dosyasının <head> içine: -->
<link rel="stylesheet" href="style.css">

2. Dahili (Internal) CSS

CSS kodları, HTML dosyasının kendi <head> bölümü içine, <style> etiketleri arasına yazılır.

Ne Zaman Kullanılır? Genellikle tek bir sayfaya özel stiller uygulamak için veya hızlı bir test yapmak için kullanılır. Harici CSS'ten daha az tercih edilir çünkü stiller diğer sayfalarda yeniden kullanılamaz.

<!-- index.html dosyasının <head> içine: -->
<style>
  body {
    background-color: lightblue;
  }
  h1 {
    color: navy;
  }
</style>

3. Satır İçi (Inline) CSS (En Kötü Yöntem)

CSS kodları, doğrudan HTML elementinin style niteliği (attribute) içine yazılır.

Neden Kaçınmalıyız?

  • HTML'i kirletir ve okunaksız hale getirir.
  • Bakımı imkansızdır (10 butonu değiştirmek için 10 etiketi de bulup düzeltmeniz gerekir).
  • CSS'in "basamaklı" yapısını bozar çünkü en yüksek önceliğe sahiptir ve ezilmesi (override) zordur.

<!-- index.html dosyasının <body> içine: -->
<h1 style="color: blue; font-size: 24px;">Bu başlık mavidir</h1>

Söz Dizimi (Syntax) ve Seçiciler

Bir CSS kuralı iki ana bölümden oluşur: Seçici (Selector) ve Bildirim Bloğu (Declaration Block).

  • Seçici (Selector): Hangi HTML elementini (veya elementlerini) stilize etmek istediğimizi belirtir (örn: h1, .kart, #ana-baslik).
  • Bildirim Bloğu (Declaration Block): { ... } süslü parantezleri arasındaki kısımdır.
  • Özellik (Property): Değiştirmek istediğimiz stil özelliğidir (örn: color, font-size, background-color).
  • Değer (Value): O özelliğe atadığımız değerdir (örn: blue, 16px, #FFF).

En Sık Kullanılan Seçiciler

Seçici Türü Örnek Açıklama
Etiket (Type) p { ... } Sayfadaki tüm <p> etiketlerini seçer.
Sınıf (Class) .kart { ... } class="kart" niteliğine sahip tüm elementleri seçer. En çok kullanılan seçicidir.
ID #ana-baslik { ... } id="ana-baslik" niteliğine sahip olan *tek* elementi seçer. ID'ler benzersiz olmalıdır.
Evrensel (Universal) * { ... } Sayfadaki **tüm** elementleri seçer. (Genellikle box-sizing sıfırlaması için kullanılır).
Gruplama h1, h2, h3 { ... } Virgülle ayrılmış tüm seçicileri (h1, h2 ve h3) aynı anda seçer.
Torun (Descendant) .icerik p { ... } .icerik sınıfına sahip bir elementin *içindeki* tüm <p> etiketlerini seçer.
Çocuk (Child) .kart > img { ... } .kart sınıfının *doğrudan alt öğesi* (çocuğu) olan <img> etiketlerini seçer.
Sözde Sınıf (Pseudo-class) a:hover { ... } Bir elementin belirli bir durumunu seçer (örn: fare üzerine gelince, tıklandığında, ilk çocuk).
Sözde Element (Pseudo-element) p::first-line { ... } Bir elementin belirli bir parçasını seçer (örn: ilk satırı, öncesi, sonrası).

Kutu Modeli (Box Model)

CSS'te her element, dört katmandan oluşan dikdörtgen bir kutu olarak düşünülür. Bu, web mizanpajının (layout) temel taşıdır.

  • 1. Content (İçerik): Kutunun içeriği (metin, resim vb.). Genişliği width, yüksekliği height ile belirlenir.
  • 2. Padding (İç Boşluk): İçerik ile kenarlık arasındaki iç boşluktur. Kutunun "iç duvarının" kalınlığıdır.
  • 3. Border (Kenarlık): Padding'i çevreleyen kenarlıktır.
  • 4. Margin (Dış Boşluk): Kenarlığın dışındaki, diğer elementlerle arasındaki dış boşluktur.
.kutu {
  width: 200px;          /* İçerik alanı genişliği */
  height: 100px;         /* İçerik alanı yüksekliği */

  padding: 20px;         /* 4 yönden de 20px iç boşluk */

  border: 5px solid black; /* 5px kalınlığında siyah kenarlık */

  margin: 30px;          /* Diğer elementlerle 30px dış boşluk */
}

En Önemli Kural: `box-sizing: border-box`

Varsayılan olarak (box-sizing: content-box), bir elemente width: 200px verdiğinizde, bu sadece *içerik* alanının genişliğidir. Üzerine padding: 20px ve border: 5px eklerseniz, elementin toplam genişliği:

200px (width) + 40px (sağ/sol padding) + 10px (sağ/sol border) = 250px olur.

Bu durum, hesaplamayı çok zorlaştırır ve istenmeyen taşmalara neden olur.

Çözüm: box-sizing: border-box; kuralı, tarayıcıya şunu der: "Sana verdiğim width: 200px değeri, o elementin *toplam* genişliği olsun. Padding ve Border'ı bu 200px'in içinden al, dışına ekleme."

Genellikle tüm elementlere bu kuralı uygulamak en iyi pratiktir:

* {
  box-sizing: border-box;
}

Birimler (Units)

CSS'te boyut ve mesafe belirtmek için çeşitli birimler kullanılır. Bunlar iki ana gruba ayrılır:

1. Mutlak (Absolute) Birimler

Boyutu sabit olan, başka bir değere göre değişmeyen birimlerdir. En yaygını px'tir.

  • px (Piksel): Ekrandaki bir pikseli temsil eder. En temel ve yaygın birimdir. font-size: 16px; veya border: 1px solid red;

2. Göreceli (Relative) Birimler

Başka bir değere (ebeveyn elementin boyutu, tarayıcı penceresinin boyutu vb.) göre hesaplanan esnek birimlerdir. Duyarlı tasarım (responsive design) için çok önemlidirler.

  • % (Yüzde): İçinde bulunduğu elementin (ebeveyn) ilgili özelliğinin yüzdesini alır. width: 50%; (Ebeveynin %50'si kadar geniş ol).
  • em: Elementin kendi font-size değerine göre hesaplanır. Eğer font-size: 16px; ise, 2em = 32px olur. İç içe elementlerde karmaşıklaşabilir.
  • rem (Root EM): **En çok tavsiye edilen** göreceli birimdir. em gibidir ancak her zaman *kök* elementin (<html>) font-size değerini baz alır. Tarayıcılarda bu varsayılan olarak 16px'tir.
    • 1rem = 16px
    • 1.5rem = 24px
    • 0.5rem = 8px

    Tüm padding, margin ve font-size değerlerini rem ile vermek, kullanıcının tarayıcı ayarlarından yazı tipi boyutunu büyütmesi durumunda tüm sitenin orantılı olarak büyümesini sağlar (Erişilebilirlik için harikadır).

  • vw (Viewport Width): Tarayıcı penceresinin genişliğinin %1'idir. 100vw = Tarayıcının tam genişliği.
  • vh (Viewport Height): Tarayıcı penceresinin yüksekliğinin %1'idir. 100vh = Tarayıcının tam yüksekliği. (Tüm ekranı kaplayan bir açılış bölümü yapmak için kullanılır).

Tipografi (Yazı Tipleri)

Web sitenizin okunabilirliği ve karakteri için tipografi ayarları çok önemlidir.

  • font-family: Yazı tipi ailesini belirler. Birden fazla değer girilerek "font yığını" (font stack) oluşturulur. Tarayıcı ilkini bulamazsa ikincisini dener. 
    font-family: "Poppins", Arial, sans-serif;
  • font-size: Yazı tipi boyutunu belirler (Örn: 16px, 1.1rem).
  • font-weight: Yazı tipi kalınlığını belirler (Örn: normal, bold, veya 100-900 arası sayılar).
  • line-height: Satır yüksekliğini belirler. Okunabilirlik için genellikle 1.5 veya 1.7 gibi birimsiz değerler kullanılır (font boyutunun 1.5 katı).
  • text-align: Metnin yatay hizalamasını belirler (left, center, right, justify).
  • text-decoration: Metnin altı/üstü/ortası çizgili olmasını belirler. Genellikle linklerin altındaki çizgiyi kaldırmak için kullanılır (text-decoration: none;).

Google Fonts (Harici Yazı Tipi) Nasıl Kullanılır?

  1. Google Fonts sitesine gidin ve bir yazı tipi (örn: Poppins) seçin.
  2. İstediğiniz kalınlıkları (örn: 400 Regular, 700 Bold) seçin.
  3. Google'ın size verdiği <link> kodunu kopyalayıp HTML'inizin <head> bölümüne yapıştırın.
  4. Verilen font-family kuralını CSS'inizde (örn: body seçicisinde) kullanın.

Renkler ve Arkaplanlar

Renk Değerleri Nasıl Belirtilir?

  • Keyword (Anahtar Kelime): red, blue, black. Sınırlı sayıda renk vardır.
  • HEX (Hexadecimal): #RRGGBB. En yaygın kullanılan yöntemdir.
    #FF0000 (Kırmızı), #000000 (Siyah), #FFFFFF (Beyaz), #333 (Koyu Gri - kısa yazım).
  • RGB (Red, Green, Blue): rgb(255, 0, 0). 0-255 arası değerler alır.
  • RGBA (Red, Green, Blue, Alpha): rgba(255, 0, 0, 0.5). RGB'ye ek olarak 0 (tamamen şeffaf) ile 1 (opak) arasında bir "Alpha" (şeffaflık) kanalı ekler.

Temel Özellikler

  • color: Bir elementin içindeki metnin rengini değiştirir.
  • background-color: Bir elementin arkaplan rengini değiştirir.
  • background-image: Arka plana bir resim ekler. 
    background-image: url('resimler/arkaplan.jpg');
  • background-size: Arkaplan resminin boyutunu ayarlar (cover: resmi sığdırıp kapla, contain: resmi bozmadan sığdır).
  • background-position: Resmin konumunu ayarlar (center, top right vb.).
  • background-repeat: Resmin tekrarlanıp tekrarlanmayacağını ayarlar (no-repeat, repeat-x vb.).

Konumlandırma (Position)

position özelliği, elementlerin normal belge akışının dışına nasıl çıkarılacağını ve nasıl konumlandırılacağını belirler. Kutu Modeli ve Flex/Grid ile birlikte mizanpajın üçüncü temel direğidir.

  • position: static; (Varsayılan): Element normal HTML akışına göre konumlanır. top, left gibi özellikler etki etmez.
  • position: relative; (Göreceli): Element normal akışa göre konumlanır, ancak top, left, bottom, right özellikleri ile normal konumuna *göre* kaydırılabilir. **En önemli özelliği, absolute konumlandırılmış çocuk elementler için bir "çapa" (konumlandırma ebeveyni) görevi görmesidir.**
  • position: absolute; (Mutlak): Element, normal akıştan tamamen çıkarılır. En yakın position: relative (veya absolute, fixed, sticky) olan ebeveynine göre konumlanır. Eğer böyle bir ebeveyn bulamazsa <body>'e göre konumlanır.
  • position: fixed; (Sabit): Element, normal akıştan çıkarılır ve tarayıcı penceresine (viewport) göre sabitlenir. Sayfa kaysa bile element hep aynı yerde kalır (örn: sabit menüler, "yukarı çık" butonu).
  • position: sticky; (Yapışkan): Element, normal akışta yer alır, ancak kullanıcı sayfayı kaydırdığında belirli bir noktaya (örn: top: 0;) ulaştığında orada fixed gibi yapışıp kalır.

z-index

position değeri static dışında olan elementler üst üste binebilir. z-index, bu elementlerin yığılma sırasını belirler. Değeri yüksek olan, düşük olanın üstünde görünür. (Sadece position tanımlanmış elementlerde çalışır).

Flexbox ile Hizalama

Flexbox, elementleri bir konteyner içinde **tek bir boyutta (yatay veya dikey)** hizalamayı, dağıtmayı ve sıralamayı inanılmaz kolaylaştıran modern bir CSS düzen modülüdür.

Flexbox'ta iki ana oyuncu vardır: Konteyner (Ebeveyn) ve Öğeler (Çocuklar).

Konteyner (Ebeveyn) Özellikleri

Stiller, display: flex; verdiğiniz ana kutuya uygulanır.

  • display: flex;: Flexbox düzenini etkinleştiren sihirli anahtar.
  • flex-direction:: Öğelerin dizilme yönünü belirler (row: yatay - varsayılan, column: dikey).
  • justify-content:: Öğeleri ana eksende (yatayda) nasıl hizalayacağını belirler.
    • flex-start: Başa yasla (varsayılan).
    • flex-end: Sona yasla.
    • center: Ortala.
    • space-between: İlk ve son öğeyi kenarlara yasla, aradaki boşlukları eşit dağıt.
    • space-around: Tüm öğelerin etrafına eşit boşluk bırak.
  • align-items:: Öğeleri çapraz eksende (dikeyde) nasıl hizalayacağını belirler.
    • stretch: Ebeveyn kadar uzat (varsayılan).
    • flex-start: Üste yasla.
    • flex-end: Alta yasla.
    • center: Dikeyde ortala.
  • flex-wrap:: Öğeler konteynere sığmadığında ne yapacağını belirler (nowrap: sığdırmaya çalışır/taşar, wrap: alt satıra indirir).

Öğe (Çocuk) Özellikleri

Stiller, display: flex; içindeki çocuklara uygulanır.

  • flex-grow:: Bir sayı alır (örn: 1). Ebeveynde boş yer kalırsa, o boşluğun ne kadarını alacağını belirler.
  • flex-shrink:: Bir sayı alır (örn: 1). Öğeler sığmazsa, ne kadar küçüleceğini belirler.
  • flex-basis:: Öğenin esnemeden/küçülmeden önceki varsayılan boyutunu belirler.
  • order:: Öğelerin sırasını değiştirir (örn: order: -1; en başa alır).

Grid ile Mizanpaj

CSS Grid, Flexbox'ın aksine **iki boyutlu (hem satır hem sütun)** bir düzen sistemi sunar. Karmaşık sayfa mizanpajları (layout) oluşturmak için tasarlanmıştır. Flexbox hizalama için, Grid ise mizanpaj için mükemmeldir.

Konteyner (Ebeveyn) Özellikleri

  • display: grid;: Grid düzenini etkinleştirir.
  • grid-template-columns:: Sütunların yapısını tanımlar. 
    /* 3 eşit sütun */
grid-template-columns: 1fr 1fr 1fr;
/* veya */
grid-template-columns: repeat(3, 1fr);

/* Bir sidebar ve bir ana içerik */
grid-template-columns: 300px 1fr;

    fr (fraction - kesir) birimi, mevcut alanın orantılı bir parçasını temsil eder.

  • grid-template-rows:: Satırların yapısını tanımlar (örn: 100px 1fr 50px; - header, content, footer).
  • gap: (veya grid-gap): Satır ve sütunlar arasındaki boşluğu belirler (örn: 20px).

Öğe (Çocuk) Özellikleri

Çocuk öğelerin grid içinde ne kadar alan kaplayacağını belirleriz.

  • grid-column-start: / grid-column-end:: Öğenin hangi sütun çizgisinde başlayıp hangisinde biteceğini belirler.
  • grid-row-start: / grid-row-end:: Öğenin hangi satır çizgisinde başlayıp hangisinde biteceğini belirler.
  • grid-column: 1 / 3; (Kısa yazım: 1. çizgiden başla, 3. çizgide bitir).
  • grid-column: span 2; (Kısa yazım: 2 sütunluk yer kapla).

Duyarlı Tasarım (Responsive Design) & Media Query

Duyarlı Tasarım, bir web sitesinin mobil telefonlardan tabletlere ve geniş masaüstü ekranlara kadar tüm cihazlarda iyi görünmesini ve düzgün çalışmasını sağlama pratiğidir.

Bunun için HTML'de etiketinin (HTML rehberinde açıklandı) olması **zorunludur**.

CSS'teki aracı ise @media sorgularıdır.

Media Query Nedir?

Media sorgusu, tarayıcı penceresinin genişliği, yüksekliği veya yönü gibi belirli koşullar sağlandığında uygulanacak özel CSS kuralları bloğudur.

"Mobile-First" (Önce Mobil) Yaklaşımı

En iyi pratiktir. Önce sitenin en dar (mobil) ekranlardaki stillerini yazarız (genellikle tek sütunlu, alt alta). Sonra @media sorguları ile ekran büyüdükçe (min-width - minimum genişlik) mizanpajı değiştiririz.

Örnek:

Önce mobil cihazlar için bir kart yapısı düşünelim:

/* Mobil stilleri (varsayılan) */
.kart-konteyner {
  display: block; /* Normal akışta alt alta */
}
.kart {
  width: 100%;
  margin-bottom: 15px;
}

Şimdi, ekran genişliği 768px'i (tablet boyutu) geçtiğinde, bu kartları yan yana göstermek için Flexbox veya Grid kullanalım:

/* Tablet ve üzeri (min-width: 768px) */
@media (min-width: 768px) {
  .kart-konteyner {
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
  }
  .kart {
    /* 20px boşluğu hesaba katarak yaklaşık %50'lik alan */
    flex-basis: calc(50% - 10px);
    margin-bottom: 0;
  }
}

/* Geniş ekranlar (min-width: 1200px) */
@media (min-width: 1200px) {
  .kart {
    /* 3'lü dizilim için */
    flex-basis: calc(33.333% - 14px);
  }
}

Bu yaklaşımla, siteniz her ekran boyutuna uyum sağlar.
